# LED 控制 APP 设计文档

## 1. 背景

当前固件已经实现了基于 ESP8266 的 MicroPython 灯带控制能力，支持两种运行模式：

- **配置模式**：设备未联网时开启热点 `LED_Config`，监听 UDP 端口 `8889`，用于接收 Wi‑Fi 凭据。
- **控制模式**：设备联网后监听 UDP 端口 `8888`，用于接收灯效与亮度控制命令。

固件当前提供的能力已经足够支持一个本地局域网内使用的 APP 首版。因此，本设计只覆盖 **APP 配网与设备控制**，不引入云端账号、多设备编排、定时任务等超出当前固件能力的内容。

## 2. 目标

首版 APP 的目标是形成一个可用闭环：

1. 发现设备
2. 完成配网
3. 进入控制页
4. 控制灯效与亮度
5. 管理设备的基础信息

## 3. 范围

### 3.1 本期包含

- 设备发现
- 手动添加设备 IP
- 配网引导
- Wi‑Fi 列表展示
- SSID 选择与密码提交
- 配网等待与重连
- 基础灯效控制
- 亮度调节
- 设备基础管理

### 3.2 本期不包含

- 云端账号系统
- 远程公网控制
- 多设备场景编排
- 定时任务
- 灯效参数细化配置
- 固件升级

## 4. 现有能力映射

APP 需要直接对接当前固件已实现的 UDP 协议。

### 4.1 配网模式

设备在未联网时进入 `config_mode()`，监听 `CFG_PORT = 8889`。
支持命令：

- `config:SSID:PASSWORD`
- `status`
- `list`

同时设备会周期性广播可见 Wi‑Fi 列表，格式为 `WIFIS:...`。

### 4.2 控制模式

设备联网后进入 `control_mode()`，监听 `CTRL_PORT = 8888`。
支持命令：

- `mode:效果名称`
- `mode:next`
- `mode:prev`
- `bright:0-255`
- `status`
- `help`

## 6. 页面设计

### 6.1 设备搜索页

**职责**
- 扫描局域网内已联网设备
- 作为进入控制或配网流程的入口

**主要元素**
- 标题
- 扫描按钮
- 设备列表
- 手动添加 IP 按钮
- 进入配网按钮

**状态**
- 扫描中
- 已发现设备
- 未发现设备

### 6.2 配网引导页

**职责**
- 提示用户连接设备热点 `LED_Config`
- 引导用户进入配网流程

**主要元素**
- 说明文案
- 热点名称展示
- 已连接热点按钮
- 查看连接步骤按钮

### 6.3 Wi‑Fi 选择页

**职责**
- 展示设备广播出来的可见 Wi‑Fi 列表
- 允许手动输入 SSID

**主要元素**
- Wi‑Fi 列表
- 刷新列表按钮
- 手动输入 SSID 按钮

**数据来源**
- 设备广播 `WIFIS:...`
- UDP `list` 响应

### 6.4 密码输入页

**职责**
- 采集 Wi‑Fi 密码并提交配网命令

**主要元素**
- 已选 SSID 显示
- 密码输入框
- 显示密码开关
- 连接按钮

**提交命令**
- `config:SSID:PASSWORD`

### 6.5 配网等待页

**职责**
- 告知用户设备正在保存配置并重启
- 处理等待与重连

**主要元素**
- 加载动画
- 状态文案
- 重新尝试按钮

**行为**
- 接收设备返回的成功信息
- 轮询设备是否重新上线
- 重新上线后跳转控制页

### 6.6 主控制页

**职责**
- 提供日常控制能力

**主要元素**
- 设备名称
- 在线状态
- 当前模式
- 亮度滑条
- 上一效果 / 下一效果按钮
- 8 个灯效按钮

**灯效按钮**
- rainbow
- breath
- fire
- starry
- wave
- chase
- sparkle
- snake

### 6.7 设备设置页

**职责**
- 管理当前设备

**主要元素**
- 名称
- IP
- 状态
- 重新配网入口
- 删除设备入口

## 7. 组件划分建议

APP 逻辑建议拆成 3 个独立层：

### 7.1 设备发现层

负责：
- 局域网扫描
- 手动添加 IP
- 在线状态维护

### 7.2 配网流程层

负责：
- 热点引导
- Wi‑Fi 列表展示
- 密码提交
- 重启后重连

### 7.3 控制层

负责：
- 模式切换
- 亮度控制
- 状态查询
- 设备信息展示

## 8. 交互与协议映射

### 8.1 设备发现

- APP 扫描局域网
- 如果发现可用设备，则请求 `status`
- 若无可用设备，则进入配网页

### 8.2 配网交互

1. 用户连接设备热点 `LED_Config`
2. APP 获取 Wi‑Fi 列表
3. 用户选择 SSID 并输入密码
4. APP 发送 `config:SSID:PASSWORD` 到 UDP 8889
5. 设备返回成功后重启
6. APP 重新搜索设备并进入控制页

### 8.3 控制交互

- 切换效果：`mode:效果名称`
- 下一效果：`mode:next`
- 上一效果：`mode:prev`
- 调整亮度：`bright:0-255`
- 查询状态：`status`

## 9. 开发优先级

### 第一阶段

- 设备搜索页
- 配网引导页
- Wi‑Fi 选择页
- 主控制页

### 第二阶段

- 密码输入页
- 配网等待页
- 设备设置页

### 第三阶段

- 自动重连优化
- 状态缓存
- 常用效果快捷入口

## 10. 验收标准

首版 APP 完成后应满足：

- 能发现局域网内已联网的设备
- 能引导用户完成热点配网
- 能通过 UDP 成功发送配网命令
- 能在设备重启后重新发现设备
- 能切换至少 8 种灯效
- 能调整亮度
- 能展示设备当前状态

## 11. 风险与约束

- 设备端运行在 ESP8266 上，内存有限，APP 交互应尽量简单直接。
- 配网依赖热点接入，首次使用时用户可能需要手动切换 Wi‑Fi。
- 当前协议仅支持本地局域网控制，不支持公网访问。

## 12. 结论

本期 APP 设计以“本地局域网内的最小可用遥控器”为目标，优先完成配网与基础控制闭环。当前固件能力已经覆盖协议基础，因此 APP 设计应保持轻量、直观，并为后续扩展保留页面层级与状态层级的独立性。
